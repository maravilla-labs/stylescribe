## What Are Tutorials?

Tutorials are **learning-oriented** — they take you through building something from scratch. Unlike how-to guides (which solve specific problems) or reference docs (which list all options), tutorials help you understand the workflow.

### Tutorial Philosophy

1. **Hands-on** — You write real code, not just read about it
2. **Complete** — You end with a working result
3. **Explained** — Each step explains *why*, not just *what*
4. **Buildable** — You can extend what you learned
